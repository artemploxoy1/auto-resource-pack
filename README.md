# Плагин Auto-Resource-Pack 📦

## Описание плагина

`Auto-Resource-Pack` — это системный сетевой плагин для BlockMine, разработанный для решения проблем совместимости протоколов, сетевых таймаутов и вылетов бота при межсерверных переходах (BungeeCord/Velocity) на современных версиях Minecraft (1.20.3 - 1.21.x+). 

Плагин оптимизирован для работы на крупных серверах со сложной структурой авторизации и перенаправлений (например, **FunTime**). Он автоматизирует скачивание ресурс-паков, правильно общается с сервером на фазе конфигурации и защищает бота от киков, вызванных рассинхронизацией физики и ошибками парсинга кодеков.

---

## Основные функции

1. **Автоматическая загрузка ресурс-паков (1.20.3 - 1.21+)**:
   Реагирует на запросы `add_resource_pack`, скачивает архивы в локальную папку плагина по HTTP/HTTPS (с поддержкой перенаправлений) и отправляет серверу правильную цепочку статусов (`ACCEPTED` $\rightarrow$ `DOWNLOADED` $\rightarrow$ `SUCCESSFULLY_LOADED`) через нативный сериализатор протокола.

2. **Совместимость с протоколом 1.21+ (select_known_packs)**:
   Автоматически отвечает на пакеты согласования известных ресурсов `select_known_packs` во время конфигурационной фазы, предотвращая немедленный кик бота прокси-сервером.

3. **Сетевой фильтр пакетов (PhysicsGuard & Firewall)**:
   При переходе в фазу конфигурации (`start_configuration`) плагин полностью отключает физический движок бота и блокирует отправку любых игровых пакетов (движение, замахи, использование предметов) до тех пор, пока бот окончательно не заспавнится в игровом мире нового сервера. Физика плавно возобновляется через 2 секунды после события `spawn`.

4. **Защита от падения Dimension Codec (Proxy-Fallback)**:
   При возникновении ошибок парсинга кодеков измерений (частая проблема при использовании ViaVersion/ViaBackwards на стороне сервера, вызывающая `TypeError` в недрах Mineflayer), плагин подменяет оригинальный реестр динамическим `Proxy`-объектом. Это позволяет безопасно возвращать дефолтные параметры мира для любых неизвестных ID измерений (например, ID `3` при нахождении в хабе) и избегать критической остановки процесса.

5. **Хотпатч пакета настроек**:
   Исправляет ошибку отправки устаревшего пакета `settings` вместо `client_information` во время фазы конфигурации.

## Системные требования и рекомендации

* **Версия BlockMine:** 1.20.0 или выше.
* **Версия протокола:** Рекомендуется устанавливать версию **1.21.1** в настройках сервера бота для наиболее стабильного прохождения фазы конфигурации на FunTime.
* **Порты:** Порт `443` должен быть открыт для исходящих HTTPS-соединений, чтобы бот мог скачивать файлы паков с удаленных серверов.

---

## Настройки плагина

Параметры настраиваются через графический интерфейс управления плагином в веб-панели BlockMine:

* **Задержка скачивания (мс)** (`simulatedDelayMs`):
  * *Тип:* `number`
  * *По умолчанию:* `100`
  * *Описание:* Имитирует задержку применения ресурс-пака после его загрузки перед отправкой серверу финального статуса успешного применения. Помогает избежать подозрений в использовании автоматизированного софта.

---

## Структура папок после запуска

После первого успешного скачивания ресурс-пака в папке плагина появится каталог для хранения кэшированных файлов:
```
/auto-resource-pack
├── resourcepacks/          # Сюда скачиваются zip-архивы паков с именами в виде их UUID или хэша
├── package.json
└── index.js
```

## Логирование событий

Плагин выводит подробную информацию о сетевых фазах в консоль бота:
* `[AutoResourcePack] [PhysicsGuard] Вход в фазу конфигурации...` — начало перехода на другой сервер. Физика приостановлена.
* `[AutoResourcePack] [1.20.3+] Получен запрос ресурс-пака...` — сервер запросил установку пака.
* `[AutoResourcePack] [Fix] Ошибка при загрузке Dimension Codec...` — зафиксирована ошибка ViaVersion, применен безопасный обход.
* `[AutoResourcePack] [PhysicsGuard] Бот успешно заспавнился. Запуск таймера...` — бот перешел на новый сервер, подготовка к запуску движений.
