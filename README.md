# Плагин Auto Resource Pack 📦

Плагин для автоматического принятия и подтверждения загрузки ресурс-паков от сервера. Спроектирован специально для бесперебойной работы ботов на крупных игровых проектах с BungeeCord/Velocity перенаправлениями.

## Описание проблемы и решение
В стандартной библиотеке Mineflayer высокоуровневый метод `bot.acceptResourcePack()` часто не успевает сработать, когда сервер принудительно переключает состояние сетевого клиента (например, при переходе из лобби авторизации в основной хаб или игру). Из-за этого сервер не получает вовремя ответ от клиента и кикает бота с ошибкой.

**Auto Resource Pack** решает эту проблему на уровне сырых сетевых пакетов (`add_resource_pack` и `resource_pack_send`), перехватывая их напрямую в протоколе, в том числе во время фазы конфигурации (`configuration state`), обеспечивая стабильную работу без дисконнектов.

---

## Как это работает
1. При отправке сервером запроса на установку ресурс-пака, плагин мгновенно отправляет пакет подтверждения `resource_pack_receive` со статусом `ACCEPTED` (3).
2. Плагин ожидает указанное в настройках время (имитация скачивания).
3. По истечении таймера плагин отправляет финальный пакет со статусом `SUCCESSFULLY_LOADED` (0).
4. Сервер считает, что бот успешно применил ресурс-пак, и не прерывает соединение.

---

## Настройки плагина
Настройки доступны в файле конфигурации или через веб-интерфейс панели управления бота:

*   **Задержка скачивания (`simulatedDelayMs`)**: 
    *   *Тип*: Число (миллисекунды).
    *   *Значение по умолчанию*: `100`.
    *   *Описание*: Время ожидания перед отправкой подтверждения об успешной установке пака. На некоторых серверах слишком мгновенный ответ (0 мс) может детектироваться античитами как подозрительный. Рекомендуется оставлять в пределах `100`–`500` мс.

---

## Для разработчиков
Плагин автоматически отслеживает пересоздание сетевого сокета при реконнектах бота (событие `login`) и перепривязывает обработчики к новому клиенту `bot._client`. 

При деактивации или удалении плагина через панель, все зарегистрированные сетевые слушатели автоматически и безопасно удаляются, предотвращая утечки памяти.
